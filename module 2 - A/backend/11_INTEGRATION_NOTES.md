# INTEGRATION NOTES\n# Module 2 - Part A: Patient Healthcare Vault\n\n## Overview\nThis module provides the backend API for patient-owned healthcare data management. INTEGRATION WITH MODULE 1 (AUTHENTICATION & AUTHORIZATION)\n\n### JWT Token Claims\nThe application expects JWT tokens from Module 1 with these claims:\n\n```json\n{\n  \"sub\": \"usr_123456\",              // User ID (required)\n  \"email\": \"user@example.com\",      // Email\n  \"roles\": [\"patient\"],             // Role array\n  \"patient_id\": \"pat_abc123def456\",  // Patient ID (for patient role)\n  \"provider_id\": \"doc_xyz789\",      // Provider ID (for provider role)\n  \"exp\": 1624056000,                // Expiration timestamp\n  \"iat\": 1624052400                 // Issued at timestamp\n}\n```\n\n### Implementation Details\n\n1. **Token Validation**\n   - All endpoints except `/health` and `/ready` require valid JWT\n   - Token extracted from `Authorization: Bearer <token>` header\n   - Validation in `dependencies.py:get_current_user()`\n\n2. **User Identification**\n   - `current_user[\"sub\"]` = User ID from auth system\n   - `current_user[\"patient_id\"]` = Patient profile ID\n   - `current_user[\"roles\"]` = User roles for RBAC\n\n3. **Authorization Logic**\n   ```python\n   # Patient accessing own data\n   if current_user.get(\"patient_id\") == patient_id:\n       allow_access()\n   \n   # Provider with consent\n   if current_user.get(\"role\") == \"provider\":\n       check_consent_records()\n   \n   # Family member with approval\n   if current_user.get(\"role\") == \"family\":\n       verify_family_access()\n   ```\n\n### Configuration\n- Set `JWT_SECRET` environment variable to match Module 1's secret\n- Set `JWT_ALGORITHM` (default: HS256)\n- Token expiration handled by Module 1\n\n---\n\n## 2.
